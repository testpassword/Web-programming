Собстна, у нас есть команда asadmin, которая у нас царь и бог.
У препода добываем portbase. В моём случае -- 10000.

1)Создадим домен в текущем каталоге:
$ asadmin create-domain --domaindir . --portbase 10000 lab7
, где domaindir -- путь до каталога с доменами (там, где вы хотите разместить свою лабу. По дефолту берётся каталог, в который студенты не имеют прав на запись, поэтому всегда нужно явно указывать свой.)
lab7 -- название лабы (с таким имененм будет создан каталог)
Примерный вывод:
Using port 10848 for Admin.
Using port 10880 for HTTP Instance.
Using port 10076 for JMS.
Using port 10037 for IIOP.
Using port 10081 for HTTP_SSL.
Using port 10038 for IIOP_SSL.
Using port 10039 for IIOP_MUTUALAUTH.
Using port 10086 for JMX_ADMIN.
Using port 10066 for OSGI_SHELL.
Using port 10009 for JAVA_DEBUGGER.

2)Запустим домен:
$ asadmin start-domain --domaindir . lab7

3)Посмотрим, что домен запустился как надо:
$ asadmin list-domains --domaindir . 

4)Посмотрим, что сервер запустился:
helios.cs.ifmo.ru:10080

5)Обламываемся
Домены глассфиш недоступны снаружи, так что...
Учимся пробрасывать порты через ssh:
ssh -f -N -L 10080:127.0.0.1:10080 s207666@helios.cs.ifmo.ru -p 2222

6)Заходим на 127.0.0.1:10080 в браузере и радуемся результату (в прикреплённом файле скриншот успешной работы глассфиша)

7)Как поработали, выключаем домен:
$ asadmin stop-domain --domaindir . lab7

Напоминаю, что "в пять утра просыпается демон Афанасьева и отстреливает всё лишнее". Такой отстрел может повредить вашу лабу.
